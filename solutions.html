Lab 2: Shell Scripting Answers

Here are the detailed answers to the practice questions.

1. How do you read user input into a variable?

Use the `read` command, for example: `read -p \"Enter your name: \" NAME`.

2. Explain the purpose of **command substitution** using `$(...)`.

Command substitution allows you to use the output of a command as a value for a variable or as part of a command line. For example, `FILES=$(ls -l)` stores the directory listing in the `FILES` variable.

3. What is the primary difference between `[ ]` and `[[ ]]` in conditional expressions?

`[[ ]]` is an extended test command that provides more advanced features in Bash, such as pattern matching (`=~`), and is generally a safer and more powerful option than `[ ]`.

4. Write a `case` statement that checks for a single character input (`y`, `Y`, `n`, `N`).

read -p "Continue? (y/n): " choice
case $choice in
  [yY])
    echo "Proceeding..."
    ;;
  [nN])
    echo "Exiting..."
    ;;
  *)
    echo "Invalid input."
    ;;
esac

5. How do you use the `&&` operator to chain commands?

The `&&` (AND) operator executes the second command only if the first command succeeds (returns an exit status of 0). Example: `mkdir mydir && cd mydir`.

6. How do you use the `||` operator to chain commands?

The `||` (OR) operator executes the second command only if the first command fails (returns a non-zero exit status). Example: `command_that_might_fail || echo \"Command failed.\"`.

7. Write a script with a function that takes a directory name and counts the number of files in it.

function count_files() {
  echo "Number of files in $1: $(ls -l "$1" | grep "^-" | wc -l)"
}
count_files "/path/to/directory"

8. How do you define a variable that is **local** to a function?

By using the `local` keyword before the variable name, for example: `local my_var=\"Hello\"`. This prevents the variable from affecting the global scope.

9. What is the purpose of the `break` command inside a loop?

The `break` command is used to immediately exit the current loop.

10. What is the purpose of the `continue` command inside a loop?

The `continue` command skips the current iteration of the loop and moves to the next one.

11. Write a script that uses a `select` loop to present a menu of options to the user.

select option in "Option 1" "Option 2" "Exit"; do
  case $option in
    "Option 1")
      echo "You chose Option 1."
      ;;
    "Option 2")
      echo "You chose Option 2."
      ;;
    "Exit")
      break
      ;;
    *)
      echo "Invalid option."
      ;;
  esac
done

12. How can you process the output of a command line-by-line?

A `while read` loop is commonly used for this. Example: `cat file.txt | while read line; do echo \"$line\"; done`.

13. How do you get the number of arguments passed to a function (not the script)?

Inside the function, the special variable `$#` refers to the number of arguments passed to that specific function call.

14. How can you capture the output of a command into a variable?

By using command substitution, for example: `my_var=$(ls -l)`.

15. What is the purpose of `trap` in a shell script?

`trap` is used to catch signals (like `SIGINT` from Ctrl+C) and execute a specified command or function before the script exits, allowing for clean-up operations.

16. Explain the use of **single quotes (`'`) vs. double quotes (`\"`)**.

Single quotes preserve the literal value of every character inside them, while double quotes allow for variable expansion (`$var`) and command substitution (`$(...)`).

17. How do you write a script that takes an arbitrary number of arguments and prints them one by one?

#!/bin/bash
for arg in "$@"; do
    echo "Argument: $arg"
done

18. What does the command `grep \"pattern\" file.txt` do?

The `grep` command searches for lines containing the specified `pattern` in `file.txt` and prints them to standard output.

19. How do you get the last modified timestamp of a file in a script?

Using the `stat` command, such as `stat -c %y file.txt`.

20. What is a \"here document\" and how is it used?

A here document (`<< EOF`) is a way to pass a multi-line string to a command from within a script, often used with commands like `cat`, `ssh`, or `ftp` to provide a block of input without needing to escape every line.
